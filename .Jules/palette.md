## 2024-05-22 - [Copy to Clipboard]
**Learning:** Developers frequently need to copy-paste code snippets from documentation. Adding a dedicated "Copy" button reduces friction and improves the developer experience significantly.
**Action:** When designing documentation pages, always include a "Copy" button for code blocks.

## 2025-10-06 - [Static Data Accessibility]
**Learning:** Static text data (like API URLs) is often treated as read-only, but users frequently need to copy it. Treating these as "copyable data points" with explicit buttons improves usability for developers.
**Action:** Identify static data fields that users likely need to copy (IDs, URLs, keys) and attach copy controls to them.
